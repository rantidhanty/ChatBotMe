# 03 - Memahami Istilah Scaffold

## Tujuan

Dokumen ini menjelaskan arti kata `scaffold` karena istilah ini sering muncul saat memulai project.

## Arti Sederhana

`Scaffold` berarti:

- membuat kerangka awal project secara otomatis
- menyiapkan file dan folder dasar
- memberi template awal supaya kita tidak mulai dari nol

Kalau dianalogikan:

- membangun rumah dari nol itu berat
- scaffold itu seperti rangka rumah awal yang sudah disiapkan

Jadi kita tinggal melanjutkan, bukan mulai dari tanah kosong.

## Arti Dalam Project Kamu

Saat saya bilang project Next.js ini masih berupa `scaffold default`, maksudnya:

- project sudah dibuat otomatis oleh tool
- file dasar sudah tersedia
- tampilannya masih bawaan
- belum ada fitur chatbot buatan kita

Jadi isi project sekarang masih seperti "kerangka awal", belum menjadi produk akhir.

## Contoh Di Project Ini

Tool `create-next-app` membuat beberapa hal secara otomatis:

- `package.json`
- `src/app/layout.tsx`
- `src/app/page.tsx`
- `src/app/globals.css`
- `public/`

Semua itu adalah bagian dari scaffold.

## Kenapa Scaffold Penting

Scaffold membantu karena:

- menghemat waktu setup
- memberi struktur awal yang benar
- mengurangi kesalahan konfigurasi dasar
- membuat kita bisa fokus ke fitur

## Apa Yang Belum Termasuk Scaffold

Scaffold bukan berarti project sudah selesai.

Biasanya yang belum ada:

- logic bisnis
- desain final
- integrasi API
- database
- auth
- fitur utama produk

Di project kamu, chatbot multi-provider belum ada. Yang ada baru kerangka app web-nya.

## Bedanya Scaffold Dengan Coding Fitur

`Scaffold`:

- menyiapkan fondasi
- membuat struktur awal
- belum menyelesaikan kebutuhan user

`Coding fitur`:

- menambah kemampuan nyata ke app
- membuat app melakukan sesuatu
- mulai membentuk produk yang benar-benar dipakai

## Kesimpulan

Kalau saya bilang:

"Saya scaffold Next.js project"

artinya:

"Saya membuat kerangka awal project Next.js beserta file dasar yang dibutuhkan."
